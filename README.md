# SECOM 반도체 공정 불량 예측

## 프로젝트 목적
반도체 공정 센서 데이터를 분석하여 불량품을 사전에 예측하는 머신러닝 모델 구현

## 사용 기술
Python, pandas, scikit-learn, imbalanced-learn(SMOTE), matplotlib

## 데이터
- 출처: UCI Machine Learning Repository (SECOM Dataset)
- 1567개 공정 기록, 591개 센서
- 양품 1463개, 불량 104개 (심각한 클래스 불균형)

## 과정
1. 결측치 50% 이상 컬럼 제거 (591 → 562개)
2. 나머지 결측치 평균값으로 대체
3. SMOTE로 불량 데이터 증강 (양품 1173, 불량 1173으로 균형)
4. RandomForest 모델 학습 (n_estimators=100)
5. 임계값 실험 (0.1~0.5) → 0.2 선택

## 실험 결과

### 모델 비교 (임계값 0.2 기준)
| 모델                    | Precision | Recall | F1     |
|---------------------|-----------|--------|-------|
| RandomForest      | 0.146      | 0.583 | 0.233 |
| LogisticRegression | 0.170     | 0.333  | 0.225 |

### 결측치 기준 비교
| 결측치 기준 | 남은 컬럼 수 | Recall | Precision | F1 |
|------------|-------------|--------|-----------|-----|
| 30% | 582 | 0.708 | 0.262 | 0.384 |
| 50% | 562 | 0.583 | 0.146 | 0.233 |
| 70% | 558 | 0.625 | 0.133 | 0.219 |

### 임계값 실험
- F1 최적 임계값: 0.25 (Precision 0.169, Recall 0.417)
- 최종 선택: 0.2 → 반도체 불량 미탐지 비용이 오탐지 비용보다 크므로 Recall 우선

## 주요 결과
- 불량 Recall: 0.583 (24개 중 14개 탐지)
- 불량에 가장 영향을 주는 센서: Attribute 60, 248, 520
- 결측치 기준을 30%로 엄격하게 적용할수록 Recall이 향상 → 노이즈가 많은 변수 제거가 모델 성능에 긍정적 영향
