# SECOM 반도체 공정 불량 예측

## 프로젝트 목적
반도체 공정 센서 데이터를 분석하여 불량품을 사전에 예측하는 머신러닝 모델 구현

## 사용 기술
Python, pandas, scikit-learn, imbalanced-learn(SMOTE), matplotlib

## 데이터
- 출처: UCI Machine Learning Repository (SECOM Dataset)
- 1567개 공정 기록, 591개 센서
- 양품 1463개, 불량 104개 (심각한 클래스 불균형)

## 과정
1. 결측치 50% 이상 컬럼 제거 (591 → 562개)
2. 나머지 결측치 평균값으로 대체
3. SMOTE로 불량 데이터 증강 (양품 1173, 불량 1173으로 균형)
4. RandomForest 모델 학습 (n_estimators=100)
5. 임계값 실험 (0.1~0.9) → 0.5 선택

## 실험 결과

### 모델 비교 (임계값 0.5 기준)
| 모델 | Precision | Recall | F1 |
|------|-----------|--------|----|
| RandomForest | 0.924 | 1.000 | 0.960 |
| LogisticRegression | 0.939 | 0.900 | 0.919 |

### 결측치 기준 비교
| 결측치 기준 | 남은 컬럼 수 | Recall | Precision | F1 |
|------------|-------------|--------|-----------|-----|
| 30% | 582 | 0.997 | 0.923 | 0.959 |
| 50% | 562 | 1.000 | 0.924 | 0.960 |
| 70% | 558 | 1.000 | 0.927 | 0.962 |

### 임계값 실험
- Recall 1.0 유지 구간: 0.1 ~ 0.5
- 최종 선택: 0.5 → Recall 1.0을 유지하는 구간 중 가장 높은 임계값
- 반도체 불량 미탐지 비용이 오탐지 비용보다 크므로 Recall 우선

## 주요 결과
- 불량 Recall: 1.000 (24개 전부 탐지), Precision: 0.924, F1: 0.960
- RandomForest가 LogisticRegression 대비 Recall에서 우세 (1.0 vs 0.9)
- 결측치 기준 30~70% 모두 유사한 성능 → 50% 선택이 합리적
- 불량에 가장 영향을 주는 센서: Attribute 60, 248, 520
